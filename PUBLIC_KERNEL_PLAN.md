# Public Kernel Plan

Goal: publish the smallest useful FDE public surface while preserving private
operating value and reserving rights aggressively.

## Public Surface

Publish only an abstract kernel:

- Concept overview for Fractal Decision Ecosystem（FDE）
- Four gates:
  - pre-execution fact check
  - scope routing
  - publication containment
  - done verification closeout
- One abstract recursive map:
  - core
  - router
  - leaf
- Rights notice and non-open-source license

## Keep Private

- Full 50-skill recursive implementation.
- Generator internals if they reveal private structure.
- `Documents/brain` pointers.
- Local filesystem paths.
- External AI route registry.
- Absorbed dialogues.
- Machine-specific runtime procedures.
- Private guarantee scripts.
- Any patent-candidate implementation detail until patent review is complete.

## Release Sequence

1. Finish defensive patent review.
2. Decide whether to file a provisional patent application.
3. Create a separate sanitized public repository or branch.
4. Run secret scan and personal-path scan.
5. Confirm `LICENSE`, `RIGHTS_NOTICE.md`, `DEFENSIVE_PATENT_REVIEW.md`, and
   public/private boundaries.
6. Get explicit human approval for the exact repository and visibility change.
7. Publish only the sanitized kernel.

## License Posture

Default posture: source-available, all rights reserved, no patent license, no
trademark license, no commercial use, no derivative works, no model training.

This is intentionally not MIT, Apache, BSD, GPL, Creative Commons, or other
open-source/open-content licensing.
