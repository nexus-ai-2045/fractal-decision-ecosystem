# Public Kernel 計画

目的: private operating value を保ち、rights を強く reserve しながら、FDE の最小限の useful public surface だけを公開候補として整える。

## 公開候補surface

公開候補は abstract kernel のみに限定する。

- Fractal Decision Ecosystem（FDE）の concept overview。
- Four gates:
  - pre-execution fact check
  - scope routing
  - publication containment
  - done verification closeout
- One abstract recursive map:
  - core
  - router
  - leaf
- Rights notice と non-open-source license。

## private に保つもの

- Full 50-skill recursive implementation。
- Private structure を reveal する generator internals。
- `Documents/brain` pointers。
- Local filesystem paths。
- External AI route registry。
- Absorbed dialogues。
- Machine-specific runtime procedures。
- Private guarantee scripts。
- Patent-candidate implementation details。

## release sequence

1. Public candidate の scope を `public-kernel/` 相当に限定する。
2. Secret scan と personal-path scan を実行する。
3. `LICENSE`、`RIGHTS_NOTICE.md`、public/private boundaries を確認する。
4. Exact repository と visibility change について explicit human approval を得る。
5. Sanitized kernel のみを publish 対象にする。

## license posture

Default posture は source-available、all rights reserved、no patent license、
no trademark license、no commercial use、no derivative works、no model
training です。

これは MIT、Apache、BSD、GPL、Creative Commons、その他の open-source /
open-content licensing ではありません。
