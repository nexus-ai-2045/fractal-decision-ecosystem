判断が散ったら、上から落とす。どこから始まっても stay / up / down / exception のどれかにする。

MECE Routing Tree

0. 例外確認 不明 / 矛盾 / Type1 / stop / collision / budget を先に拾う
1. 草薙 意味・違和感・方向が未確定なら問いにする
2. 秘書 情報が多すぎるなら 1-5 行 / A-B-C / 未決へ圧縮
3. Coord owner / route / stale / return_path を決める
4. Lane Foundation / Sales / Content / Research / Lab の成果物へ落とす
5. 現場操作 command / file / test / browser / cmux を diff・log・test で閉じる

Any-Start Rule

stay

今いる層で evidence / next_action まで閉じられる。

up

意味、判断材料、owner が足りない。草薙 / 秘書 / Coord へ上げる。

down

実作業や具体操作に落とせる。lane / 現場操作へ下げる。

exception

通常処理で解けない。例外表へ落として reroute する。

Exception Priority

優先 例外 条件 落とし先
1 user-stop CEO が stop / 保留 / 今やらないと言った park / discard
2 type1-or-external 外部送信 / 公開 / 削除 / 課金 / auth / prod DB / main push CEO 判断
3 unknown-or-conflict source / scope / owner / safety が不明、SSOT が矛盾 fact-check / scope-routing gate
4 emergency-repair 主 Outcome / 安全 / データを守れない Coord + 該当 lane
5 collision-risk same file / DB / queue / lane を複数主体が触る Coord で owner を 1 つに絞る
6 3x3-exhausted 3 視点 x 3 レイヤーで解けない scope / layer / owner / workspace を切り直す
7 freeze-or-budget weekly / token / 維持モード / 5/15 freeze が危険 park-retake / pointer 1 行
